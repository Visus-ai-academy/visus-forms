Primeira auditoria de seguranca completa realizada em 2026-03-25.

**Why:** Leonardo solicitou auditoria completa do projeto Visus Forms para identificar vulnerabilidades antes de ir para producao.

**How to apply:** Usar como baseline para verificar correcoes. Os 5 criticos (IDOR em rules, members, settings, theme + credenciais fracas) devem ser verificados primeiro em futuras revisoes.

Principais achados:
- 5 rotas com IDOR (rules PATCH/DELETE, members PATCH/DELETE, settings PATCH, theme PATCH, workflows PATCH/DELETE)
- NEXTAUTH_SECRET fraco e credenciais em .env
- Middleware nao cobre rotas /api/*
- Upload publico sem rate limiting e sem content inspection (magic bytes)
- Campo customCss aceita qualquer string sem sanitizacao
- Submissao publica aceita metadata/respondent sem validacao Zod
- Sem security headers no next.config.ts
